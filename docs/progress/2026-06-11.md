## 4. Pipeline architecture and protocol — why we set it up this way

Unbreakable rules to build a competition-grade, **leakage-free and reproducible** pipeline:

1. **Fixed folds:** generated once via `make_folds()` (`data/processed/folds.csv`). `StratifiedKFold(5, shuffle, seed=42)`, stratified on target deciles (`qcut(y,10)`) — so the ceiling mass does not break the folds. **Never re-split** (stacking validity depends on this).
2. **Determinism:** `seed_everything(42)` everywhere.
3. **No leakage:** every fitted transform (TF-IDF, target encoding, OOF meta, kNN-target, scaler) is fit **only on that fold's train portion**.
4. **Clip:** all predictions are clipped to [0, 100] before scoring and before writing.
5. **Artifacts:** each experiment produces `oof_{exp}.npy`, `test_{exp}.npy`, `params_{exp}.json`, `importance_{exp}.csv` → saved for blend/stack.
6. **Logging:** each run appends one row to `results.csv` + one summary to `EXPERIMENTS.md`.
7. **Device-aware:** XGBoost/CatBoost/torch use the GPU, falling back silently to CPU otherwise; LightGBM stays on CPU (fast enough at 10k rows).

**Single entry point:** everything runs through `./run.sh`, with every setting in the experiment's `configs/<exp_id>.yaml`. (`./run.sh exp005`, `./run.sh blend`, `./run.sh submit blend`, `./run.sh exp010 --hpo-trials 200` ...)

## 5. The models we used — which one, why

| Model | Why we chose it |
|---|---|
| **LightGBM** | Fast, native categorical, seconds at 10k rows; the backbone for baseline and kitchen-sink. |
| **CatBoost** | Ordered target statistics + native cats → best single GBM at 10k rows (CV 76.25). |
| **XGBoost** | Algorithmic diversity (a decorrelated member for the blend). |
| **Ridge / Logistic (on top of TF-IDF)** | Capturing text with a linear model beats SVD on full-vocab TF-IDF; fed to GBMs as an OOF meta-feature. |
| **Torch MLP** | Tabular + raw embedding; weak alone but adds **decorrelated** diversity to the blend (~11–13% weight). |
| **BERT / XLM-R (fine-tune)** | Since the text verbalizes the score, a fine-tuned Turkish transformer is plausibly the strongest single text component. |
| **Two-stage (classifier + regressor)** | `p·100 + (1−p)·reg` to explicitly model the ceiling (AUC 0.93). |

**Blend:** fold-safe **positive-Ridge stacker** (on the OOF matrix, cross-fit). Since a bad member gets ~0 weight, adding members cannot hurt the blend; it is also compared against the weight-blend and the one with lower OOF MSE is selected.

## 7. Text models and the BERT adventure (crash → fix)

Since the text verbalizes the score word for word, our biggest hope was here; but the road was rocky:

1. **The first BERT (exp009) crashed: CV MSE 3096.94 (RMSE 55.65).** Cause: randomly initializing the regression head and training it on the **raw 0–100 scale** for 3 epochs / lr 2e-5 — the head could not climb to that scale.
2. **Fix 1+2:** normalize the target to `y/100` + add a linear **warmup+decay** schedule. → **RMSE 11.92** (CV 142.07). The crash was solved.
3. **Fix 3+4 (tonight):** select the best epoch with an **inner-val split** inside the fold + 8 epochs + max_len 192. → **CV 135.07 (RMSE 11.62)**. The `MISSING: classifier.weight` warning in the logs is **normal** (we intentionally start the regression head from scratch).

**Important nuance:** BERT looks "weak" alone compared to the GBMs (CV ~135 vs ~76) **but because it is completely decorrelated**, it contributes far beyond its weight in the blend. It was exactly the fixed BERT + tuned LGBM that brought the blend from 74.93 → 74.17.

## 8. Blend strategy

- **`scripts/blend.py`:** automatically collects all `oof_*.npy` members on disk; tries (a) non-negative weight-blend via SLSQP + hill-climb, (b) fold-safe **ridge stacker**, and selects whichever has the lower OOF MSE.
- **Results:**
  - blend v1 (8 models, no embedding/BERT): **CV 74.93** → LB 84.74
  - blend v2 (11 models, +fixed exp009 + tuned exp010): **CV 74.17** → LB 83.62
  - blend v3 (with improved exp009): **CV 74.15** → LB 83.74 (within the noise, no real gain)
- The ridge stacker beat the weight-blend every time (74.15 against 74.38).

## 10. Residual analysis — is the signal exhausted? (Yes, on tabular)

`reports/eda/residual_analysis.py` examined the error of the best single model (exp010):

- **No linear signal left:** every engineered feature correlates with the residual at **|ρ| < 0.05**. There is no simple formula we are missing → **stop chasing feature interactions.**
- **No year bias** (per-year residual mean ≈ 0): the year features already absorbed the drift → year-based LB calibration will not help.
- **The ceiling is deliberately under-predicted:** for y=100 rows the bias is +4.07 (the model says ~96), but `%pred≥95 = 8.9%` already matches `%y=100 = 7.7%`. **Under MSE this is optimal.**
- **Tried and rejected:** isotonic calibration (74.62, worse), hard-snap-to-100 (no gain), LGBM meta-stacker (75.08, worse than ridge). **The blend is already MSE-optimal for this feature set.**

**Conclusion:** we hit the wall on the tabular side. The only open door is **text representation power.**
